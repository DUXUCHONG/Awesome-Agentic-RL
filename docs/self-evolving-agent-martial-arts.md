# AI 练武记：Self-Improving Agent 的五层武学体系

> Self-Evolving Agents 的五个关键技术台阶

过去几年，很多人讨论 AI Agent 的能力边界。但如果你仔细看最近一年的研究，会发现一个更重要的问题正在被提出：

**Agent 不只是要更聪明，而是要能持续变强。**

换句话说：AI 不应该只是一个训练好的模型，它应该更像一个 **可以不断修炼的系统**。

## 第一关：经验从哪里来

### [DreamGym](https://arxiv.org/abs/2511.03773)

Agent RL 最大的问题，其实不是算法，而是 **经验成本**。

传统 RL 环境非常友好——游戏、机器人、模拟控制任务——这些环境的共同特点是交互成本极低。但 Agent 的世界完全不同。真实任务往往包含网页操作、工具调用、多轮推理、API 交互，一次完整任务可能需要几十步甚至上百步交互。

这意味着：**真实 rollout 极其昂贵。** 于是 Agent 训练会遇到一个非常现实的瓶颈：经验规模上不去。

DreamGym 的关键洞察是：

> Agent 学习缺的不是算法，而是 **经验供应**。

所以它构建了一个 **experience synthesis system**——不只是模拟状态转移，而是同时生成新任务、状态变化和反馈信号。DreamGym 不只是 replay buffer，它更像一个 **经验工厂**。

🥋 **武学类比：DreamGym 就像一个练功房。** 在这里可以不断模拟对练、反复试招、快速积累战斗经验。一旦经验可以规模化生成，Agent 学习的瓶颈就开始松动。

---

## 第二关：经验如何进入训练

### [Agent Lightning](https://arxiv.org/abs/2508.03680)

DreamGym 解决的是经验问题，但现实系统还有一个更麻烦的问题：**很多 Agent 根本无法训练。**

原因很简单——现实中的 Agent 通常是复杂工程系统（LLM + 工具调用 + workflow + 多 agent 协作），这些系统通常是为了 **执行任务** 设计的，而不是为了 **训练** 设计的。于是出现一种很奇怪的情况：系统可以完成任务，但却很难把执行过程变成训练数据。

Agent Lightning 提出一个核心思想：**把"实战"和"练功"分开。** Agent 执行系统和训练系统应该是两个独立层。

具体来说，它会：
- 把复杂的 Agent 执行轨迹拆解成可训练的 transition
- 解决 credit assignment——到底是哪一步决策导致了成功或失败

这件事意义非常大，因为它让一件事情变得可能：**几乎任何 Agent 系统，都可以接入 RL 训练。**

🥋 **武学类比：很多高手每天在江湖上实战对敌。** 但如果门派没有一个体系，把这些战斗记录下来、拆解动作、复盘得失，那么这些经验就很难变成真正的武功。Agent Lightning 做的就是：把实战记录变成可以练习的招式。

---

## 第三关：经验如何变成能力

### [SkillRL](https://arxiv.org/abs/2602.08234)

当经验规模扩大后，一个更深的问题开始出现：**经验很多，但 Agent 不一定更强。**

原因在于原始轨迹通常非常混乱——一条任务轨迹里可能包含冗余步骤、噪声决策、偶然成功。如果直接用这些数据训练，Agent 很难学到稳定能力。

SkillRL 的核心洞察是：

> Agent 学到的应该是 **技能**，而不是 **轨迹**。

于是论文提出 **SkillBank**——系统会从大量经验中自动抽取可复用行为模式，并存入技能库。之后 Agent 在任务中可以调用技能、组合技能、学习新技能。

这一步非常关键，因为它改变了 Agent 学习的对象，从学习单条轨迹变成学习 **可复用行为结构**。

🥋 **武学类比：这一步就像是总结招式。** 从无数次战斗记录中，提炼出真正有效的套路。

---

## 第四关：技能如何参与进化

### [SAGE](https://arxiv.org/abs/2512.17102)

很多 Skill Library 方法都有一个共同问题：**技能只是被存储，而不是被优化。** 换句话说——技能存在，但没有进入训练闭环。

SAGE 的关键贡献是把技能 **真正放进 RL 训练循环里**。它做了两件非常关键的事情：

1. **任务序列训练** — Agent 会在一系列相似任务中连续执行，让前面的经验生成技能，并在后续任务中复用。
2. **技能进入奖励函数** — 系统不仅奖励任务完成，还奖励有效的技能使用。

于是训练目标发生变化：Agent 不只是完成任务，还在不断改进自己的技能库。这一步让系统第一次真正具备 **技能层面的自我进化**。

🥋 **武学类比：这一步就像是不断打磨招式。** 同一套武功，在不同战斗中反复使用、不断改进。

---

## 第五关：一个 Agent 不够

### [Group-Evolving Agents](https://arxiv.org/abs/2503.00000)

当系统开始自我进化后，新的瓶颈出现了：**单个 Agent 的探索能力有限。** 策略空间太大，一个 Agent 很难探索全部可能。

Group-Evolving Agents 的解决方案非常直接：**让一群 Agent 一起进化。** 不同 Agent 可以探索不同策略，然后共享经验和技能。于是系统从单 Agent 学习变成 **群体进化**。

🥋 **武学类比：这更像是门派式修炼。** 不同弟子探索不同武功路线，然后共享心得。

## 一个正在发生的范式变化

这条路线揭示了一个更深的变化。

过去 AI 系统的结构是：

```
数据 → 训练模型 → 部署
```

而 Self-Evolving Agent 的结构更像：

```
经验 → 技能 → 策略 → 持续进化
```

这意味着 Agent 不再是一次训练完成的模型，而更像一个 **不断修炼的系统**。

如果这条路线继续发展，未来的 Agent 很可能会像这样：

- 它们有经验生成系统
- 有技能库
- 有持续训练循环
- 甚至有群体协作进化

换句话说：**Agent 不再只是被训练出来的，它们会自己成长。**
